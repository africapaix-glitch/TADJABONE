# Contribuer à TADJABONE

Merci de votre intérêt pour améliorer TADJABONE ! Ce guide explique comment contribuer de manière efficace et respectueuse.

1. Ouvrir une issue
- Avant de commencer une nouvelle fonctionnalité ou correction, ouvrez une issue décrivant le problème ou la proposition. Fournissez des étapes pour reproduire, captures d'écran, et tout contexte utile.

2. Créer une branche
- Créez une branche depuis main nommée de façon claire : feature/<description> ou fix/<description>.
- Exemple : feature/inscriptions-multi-steps ou fix/bug-inscription-date

3. Écrire des commits clairs
- Utilisez des messages de commit descriptifs et en anglais ou français clair.
- Format recommandé : "type(scope): courte description" (ex. feat(ui): add mobile QR modal)

4. Tests et qualité
- Ajoutez ou mettez à jour les tests lorsque c'est pertinent.
- Respectez les conventions de style (formatage avec Prettier/ESLint si présents).

5. Ouvrir une Pull Request
- Ouvrez la PR vers la branche main et décrivez : ce que vous avez fait, pourquoi, et comment tester.
- Référencez les issues concernées avec des #numéro.

6. Revue de code
- Les mainteneurs feront la revue ; répondez aux commentaires et mettez à jour la PR.
- N'hésitez pas à demander de l'aide si vous êtes bloqué.

7. Sécurité
- Pour signaler une faille de sécurité, évitez les issues publiques. Contactez le mainteneur directement.

8. Merci
- Merci de contribuer et d'aider à rendre TADJABONE meilleur pour tous !
